## ここにワークショップ全体の概要などを書く

### なんとなくの自分のイメージ(ここは最終的には変更するので、メモがわりに書いています)

- ワークショップの概要
- [環境構築](/1-環境構築/setup)
  - 参加者に EC2 を構築してもらう
- [コントリビュートの流れとルール](/2-コントリビュートの流れとルール/contribution-flow-rule)
  - 一番最初に issue に関する説明があるが、ワークショップの時間効率的に**まず最初に AWS CDK 本家をクローンしてビルドを走らせたい**ため、issue を省略して[フォークからビルドまで](/3-フォークからビルドまで/fork-build)を説明
    - 実際に参加者が**フォーク・クローン・ビルドまで**を各自の環境で手を動かして実行(10〜15 分くらい時間かかる)
  - ビルド中に、飛ばした issue の説明と、残りのこの項目(他のビルドコマンド・テストコマンド・PR 提出方法)を説明
    - ここでは**参加者はコード変更・テスト実行などのコマンドを手元で実行は「せず」**、聞くだけ(講義ベース)
    - 実際に手を動かすのはガイド PR にて
- ガイド PR 提出
  - ここで講師側で VSCode 画面を共有し、参加者にも手を動かしてもらう
    - issue リンク飛んで読む
    - 対応案・概要などを説明？
    - ブランチ切る
      - 先ほどのクローン説明でブランチ切ったのでそれを使ってもよし、新たに切ってもよし
    - 該当コード・モジュールの説明
    - コード変更
    - ビルド
    - unit / integ テスト
    - README 変更
    - PR 提出
- 実践課題
- 実践課題の解説

AWS CDK Workshop へようこそ！

このようにコードを書いたり

```ts
// lib/my-app-stack.ts
export class MyAppStack {
  constructor(scope: Construct, id: string, props: StackProps) {
    super(scope, id);

    new Bucket(this, 'Bucket', {
      removalPolicy: RemovalPolicy.RETAIN,
      autoDeleteObjects: true,
    });
  }
}
```

このように diff を表示したり

```diff lang="ts"
// lib/my-app-stack.ts
export class MyAppStack {
  constructor(scope: Construct, id: string, props: StackProps) {
    super(scope, id);

    new Bucket(this, "Bucket", {
-     removalPolicy: RemovalPolicy.RETAIN,
+     removalPolicy: RemovalPolicy.DESTROY,
      autoDeleteObjects: true,
    });
  }
}
```
